# Gym Clicker (Качалка Кликер)

Telegram Mini App — кликер с прокачкой, Crash и Слотами, лидербордом на SQLite.

## Файлы проекта

- `main.py` — весь бэкенд (Telegram-бот + веб-сервер + HTML/JS игра внутри)
- `requirements.txt` — зависимости Python
- `Dockerfile` — сборка контейнера
- `render.yaml` — блюпринт для деплоя на Render.com одним кликом
- `README.md` — эта инструкция

Скрытых файлов (dotfiles) в проекте нет специально — все файлы видны на iPhone/в любом файловом менеджере.

## Деплой на GitHub

1. Создай новый репозиторий на GitHub (например `gym-clicker`)
2. Залей туда все файлы из этого архива (main.py, requirements.txt, Dockerfile, render.yaml, README.md)
3. Готово — репозиторий готов для подключения к Render

## Деплой на Render.com

### Вариант 1: через Blueprint (render.yaml) — проще всего
1. Зайди на https://dashboard.render.com
2. New → Blueprint
3. Подключи свой GitHub-репозиторий с этим проектом
4. Render сам найдёт `render.yaml` и предложит создать сервис `gym-clicker`
5. При создании укажи переменные окружения:
   - `BOT_TOKEN` — токен твоего бота от @BotFather
   - `GAME_URL` — можно оставить пустым на первый раз, задашь после первого деплоя
6. Нажми Apply — Render соберёт Docker-образ и задеплоит

### Вариант 2: вручную
1. New → Web Service
2. Подключи GitHub-репозиторий
3. Environment: **Docker** (Render сам увидит `Dockerfile`)
4. Plan: Free (или платный, если нужен диск, который не сбрасывается)
5. Добавь переменные окружения:
   - `BOT_TOKEN`
   - `GAME_URL`
6. Create Web Service

### После первого деплоя
Render выдаст тебе адрес вида `https://gym-clicker-xxxx.onrender.com`.

1. Зайди в Settings сервиса → Environment
2. Впиши в `GAME_URL` этот адрес (с `https://`)
3. Сохрани — сервис автоматически передеплоится с правильным URL

Теперь кнопка "Играть" в боте будет открывать твою мини-игру.

## Важно про PORT

Render сам передаёт контейнеру переменную `PORT` — не нужно задавать её вручную в настройках. Код уже читает `os.environ.get("PORT", 8080)`.

## Важно про базу данных (лидерборд)

На бесплатном плане Render файловая система эфемерна: при каждом передеплое/рестарте контейнера `leaderboard.db` создаётся заново (таблица пересоздаётся при старте — см. `init_db()`), и лидерборд обнуляется. Если нужен постоянный лидерборд:
- подключи Render Disk (Persistent Disk) к сервису и укажи путь через переменную `DB_PATH` (например `/data/leaderboard.db`), либо
- перейди на внешнюю БД (Postgres на Render и т.п.) — потребует правки кода.

## Локальный запуск

```bash
pip install -r requirements.txt
BOT_TOKEN=xxx GAME_URL=http://localhost:8080 python main.py
```

## Как играть (для описания бота)

1. Тапай на монету с бицепсом — получай "бицушки"
2. Прокачка → Удар — увеличивает доход за клик
3. Прокачка → Доход — пассивный доход в час (инвестиции в страны)
4. Краш — рискованная игра на множитель
5. Слоты — крути барабаны, выигрывай бицушки
6. Профиль — скины, ачивки, статистика, смена ника
